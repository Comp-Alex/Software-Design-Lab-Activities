# Software Design Lab Activities
# Introduction 
- Software design is a critical phase in the development lifecycle, serving as the blueprint that transforms abstract requirements into structured solutions. This repository has been created to compile and organize lab activities focused on software design principles, methodologies, and practical applications.

# Objectives
- The purpose of this repository is to:
- Provide students with hands-on exercises that reinforce theoretical concepts.
- Demonstrate how design models such as Data Flow Diagrams (DFDs), Unified Modeling Language (UML), and architectural patterns are applied in real-world scenarios.
- Encourage systematic problem-solving by breaking down complex systems into manageable components.
- Serve as a reference hub for collaborative learning, documentation, and project development.

# Lab Activities
By engaging with these activities, learners will gain practical experience in analyzing requirements, modeling system behavior, and designing maintainable, efficient, and scalable software solutions. This repository is not only a collection of exercises but also a foundation for cultivating critical thinking and professional design practices in software engineering.
